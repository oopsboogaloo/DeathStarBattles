## Features

- [x] **Irregular asteroid shapes (rotating convex polygons)** —
  - **Shape generation**: each asteroid is a convex polygon with 6–10 vertices, generated by sampling random angles (sorted) and radii around the centre point, then computing the convex hull. Store the normalised vertex offsets on the Planet object (`asteroid.vertices: Vec2[]`, unit-radius polygon).
  - **Rotation**: assign each asteroid a random `rotationSpeed` in 0.1–0.8 deg/frame, biased toward low values (e.g. `0.1 + rand * rand * 0.7`). Track `rotation` angle on the planet. Only advance rotation during FIRING and RESULTS phases, not AIMING (asteroids are still while the player aims).
  - **Rendering**: `PlanetRenderer._drawRocky` checks if `planet.type === ASTEROID` and has vertices — if so, draw a filled rotated polygon instead of a circle. Fill with the existing solid colour, no outline or gradient. Drawn live each frame (not baked into the static background canvas).
  - **Collision (SAT)**: replace the circle-circle check for asteroids with circle-vs-convex-polygon SAT. Each frame, compute the world-space rotated vertices (`rotatedVerts`). Test: for each polygon edge, project the bullet circle and all polygon vertices onto the edge normal; check for separating axis. If any axis separates, no collision. Fallback to the existing circle radius check for all other planet types.
  - **Gravity**: unchanged — use `planet.mass = radius² × density` as currently implemented, treating the asteroid as equivalent to its bounding circle.
  - **Scope**: only `PlanetType.ASTEROID` gets polygonal treatment. All other types remain circles.

- [x] **Destructible asteroids** — hitting an asteroid destroys it and replaces it with 2–4 smaller child asteroids, or nothing if it is too small.
  - **Fragmentation**: on bullet impact, `planet.destroyed = true` is set in `PhysicsEngine._handlePlanetImpact`. After each frame's physics batch, `GameLoop._processAsteroidFragments()` removes destroyed asteroids (mutating `gs.planets` in-place so `renderer._planets` stays in sync) and spawns children.
  - **Child placement**: N=2–4 children chosen randomly. Child radius = parent × factor (0.42 / 0.35 / 0.30 for N=2/3/4). Centers placed randomly within the parent bounding circle (uniform-in-disk sampling) with rejection until no two child circles overlap. If a child can't be placed after 120 attempts it is skipped.
  - **Minimum size**: parent radius < 5 game units → destroyed with no children. With typical child factors this gives at most 2 generations of fragmentation from a 20-unit asteroid.
  - **Child properties**: fresh random polygon (6–10 verts), random rotation and rotation speed, inherit parent density. `_rotatedVerts` pre-computed immediately so rendering and SAT collision are correct from the first frame.
  - **Gravity**: children are full `PlanetType.ASTEROID` planets added to `gs.planets`, so they exert gravity on bullets as normal.
  - **Rendering**: destroyed asteroids are hidden (`planet.destroyed` guard in renderer) before children appear, preventing a one-frame ghost.
