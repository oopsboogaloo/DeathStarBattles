## Features

- [x] **Irregular asteroid shapes (rotating convex polygons)** —
  - **Shape generation**: each asteroid is a convex polygon with 6–10 vertices, generated by sampling random angles (sorted) and radii around the centre point, then computing the convex hull. Store the normalised vertex offsets on the Planet object (`asteroid.vertices: Vec2[]`, unit-radius polygon).
  - **Rotation**: assign each asteroid a random `rotationSpeed` in 0.1–0.8 deg/frame, biased toward low values (e.g. `0.1 + rand * rand * 0.7`). Track `rotation` angle on the planet. Only advance rotation during FIRING and AIMING phases, not RESULTS.
  - **Rendering**: `PlanetRenderer._drawRocky` checks if `planet.type === ASTEROID` and has vertices — if so, draw a filled rotated polygon instead of a circle. Fill with the existing solid colour, no outline or gradient.
  - **Collision (SAT)**: replace the circle-circle check for asteroids with circle-vs-convex-polygon SAT. Each frame, compute the world-space rotated vertices (`rotatedVerts`). Test: for each polygon edge, project the bullet circle and all polygon vertices onto the edge normal; check for separating axis. If any axis separates, no collision. Fallback to the existing circle radius check for all other planet types.
  - **Gravity**: unchanged — use `planet.mass = radius² × density` as currently implemented, treating the asteroid as equivalent to its bounding circle.
  - **Scope**: only `PlanetType.ASTEROID` gets polygonal treatment. All other types remain circles.
